# DAIv3 Release Notes

**Version:** 1.0.0  
**Release Date:** March 8, 2026  
**Status:** Production Release - MVP Complete  

---

## Overview

DAIv3 version 1.0.0 marks the completion of the core MVP - a comprehensive distributed AI system for Windows 11 Copilot+ devices with full local-first operation, NPU/GPU/CPU hardware acceleration, intelligent knowledge management, and autonomous agent orchestration.

**Platform:** Windows 11 Copilot+, .NET 10  
**Target Hardware:** NPU (primary), GPU (fallback), CPU (final fallback)  
**Total Requirements Delivered:** 203  
**Test Coverage:** 2,300+ passing tests  

---

## Highlights

### 🎯 Core Achievements

- **Local-First AI Execution**: Complete offline capability with local model execution via Microsoft Foundry Local
- **Hardware Acceleration**: NPU-optimized embedding generation with automatic GPU/CPU fallback
- **Two-Tier Vector Search**: Fast topic-level search (<10ms) with fine-grained chunk-level refinement
- **Intelligent Task Orchestration**: Priority-based model queue with smart batching and dependency resolution
- **Learning Memory System**: Self-correction with knowledge back-propagation and scope-based promotion
- **Real-Time Transparency**: Live dashboards for queue status, indexing progress, agent activity, and system health
- **Zero External Dependencies**: Fully self-contained operation without Docker, external databases, or mandatory network access

## Technical Specifications

### Build Environment
- **Target Framework:** .NET 10 (net10.0 / net10.0-windows10.0.26100)
- **Solution File:** Daiv3.FoundryLocal.slnx
- **Build System:** MSBuild / SDK-style projects
- **Package Manager:** NuGet

### Runtime Requirements
- **Operating System:** Windows 11 (build 26100 or later)
- **Runtime:** .NET 10 Runtime
- **Hardware (Recommended):** 
  - CPU: arm64 (Snapdragon X) or x64 (Intel Core Ultra) with NPU
  - RAM: 16 GB minimum, 32 GB recommended
  - Storage: 10 GB for models and indexes
  - GPU: DirectML-compatible (optional, for fallback)

### Key Dependencies
- **Microsoft.ML.OnnxRuntime.DirectML** - NPU/GPU acceleration
- **Microsoft.ML.Tokenizers** - Token-based text processing
- **Microsoft.Data.Sqlite** - Local persistence
- **Microsoft.Extensions.AI** - AI abstraction layer
- **AngleSharp** - HTML parsing
- **ReverseMarkdown** - HTML to Markdown conversion
- **PdfPig** - PDF text extraction
- **DocumentFormat.OpenXml** - DOCX processing
- **.NET MAUI** - Cross-platform UI framework

### Performance Characteristics
- **Tier 1 Search:** 8-25ms for 10,000 vectors (CPU)
- **Tier 2 Search:** ~100-200ms for top-3 candidates
- **Embedding Generation:** <2000ms latency on CPU
- **Model Switch:** ~1-3 seconds (varies by model size)
- **Learning Retrieval:** <150ms timeout budget

## Known Issues & Limitations

### Phase 1 Limitations
1. **Topic Summarization:** Using TF-based extractive summarization pending Foundry Local SDK integration for Phase 2 SLM-based abstractive summarization
2. **Model Execution:** Some Foundry Local operations blocked pending SDK finalization

### Performance Notes
1. **CPU Performance:** While fully functional, CPU-only devices experience slower embedding generation (~1-2 seconds per document) compared to NPU devices (~100-300ms)
2. **Large Document Sets:** Initial indexing of >10,000 documents may take several minutes on CPU
3. **Model Switching:** Frequent model switches can impact throughput; batching minimizes this

### UI Limitations
1. **Dashboard Refresh:** Default 3-second refresh interval may be adjusted based on system performance
2. **Timeline View:** Deferred from CT-REQ-011 Project Master Dashboard (5/6 views implemented)

## Migration Notes

**Initial Installation - No Migration Required**

This is the initial release of DAIv3. For fresh installations:

1. Install .NET 10 Runtime
2. Deploy application binaries
3. Run first-time initialization: `daiv3 system init`
4. Configure watched directories: `daiv3 settings set watched_directories "[\"path1\",\"path2\"]"`
5. Initialize knowledge index: `daiv3 knowledge load-index`
