# 14. Glossary - Requirements

## Overview
This document specifies requirements derived from Section 14 of the design document. It defines required terminology for consistent system documentation and UI.

## Goals
- Ensure consistent terminology across UI, docs, and logs.
- Provide a shared vocabulary for implementation planning.

## Functional Requirements
- GLO-REQ-001: The system SHALL define and use the following terms consistently: Chunk, Embedding, Foundry Local, MCP, NPU, ONNX Runtime, Learning Memory, RAG, SLM, Tier 1 / Tier 2, TensorPrimitives.
- GLO-REQ-002: UI labels and documentation SHALL align with glossary definitions.
- GLO-REQ-003: The glossary SHALL be versioned and updated as new concepts are introduced.

## Non-Functional Requirements
- GLO-NFR-001: Glossary updates SHOULD be backward compatible with existing docs.

## Data Requirements
- GLO-DATA-001: Glossary entries SHALL include term, definition, and related terms.

## Acceptance Criteria
- GLO-ACC-001: All public documentation uses glossary terms consistently.
- GLO-ACC-002: The glossary is accessible from the documentation set.

## Out of Scope
- Localization of glossary terms for v0.1.

## Risks and Open Questions
- Define the owner for glossary governance.
